# 아이템 검색 결과 조회 (페이지네이션)

이미 생성된 검색 세션(`searchKey`)의 결과를 페이지 단위로 다시 조회한다. [아이템 검색 (툴팁 포함)](./아이템-검색-tooltip.md) `POST` 요청으로 검색 세션을 만든 뒤, 2페이지 이후로 넘기거나 정렬을 바꿀 때 사용한다.

```
GET /v1/market/web/items/searches/{searchKey}/tool-tip
```

| 항목 | 값 |
| --- | --- |
| 메서드 | `GET` |
| 경로 | `/v1/market/web/items/searches/{searchKey}/tool-tip` |

---

## POST와의 관계

| 구분 | `POST .../searches/tool-tip` | `GET .../searches/{searchKey}/tool-tip` |
| --- | --- | --- |
| 역할 | 검색 세션 **생성** | 생성된 세션 **재조회** |
| 검색 조건(`worldId`, `filters`) | Body로 전달 | **전달 안 함** (세션에 저장됨) |
| `searchKey` | 응답으로 **발급** | 경로에 **사용** |
| 페이지/정렬 변경 | — | 쿼리 파라미터로 지정 |
| **일일 검색 횟수 소진** | **1회 소진** | **소진 안 함** |

- 웹 거래소에서 **처음 검색 버튼을 누를 때** `POST`가 한 번 나가고, 서버가 검색 조건을 저장하며 `searchKey`(UUID)를 발급한다.
- 이후 **다음 페이지로 넘기거나 정렬을 바꿀 때**는 이 `GET`이 나간다. 필터 조건은 서버가 `searchKey`에 묶어 기억하므로, 경로의 키와 몇 가지 쿼리 파라미터만 넘기면 된다.

> 💡 **검색 횟수 절약**: 일일 검색 한도(`GET /v1/market/web/daily-limit`의 `search.limit`, 관측값 100회/일)는 **`POST`만 1회 소진**하고 **`GET`은 소진하지 않는다.** (실측 2026-07-08: 잔여 `91 → POST → 90 → GET → 90`) 따라서 한 번 `POST`로 세션을 만든 뒤에는 `searchKey`로 여러 페이지를 `GET`해도 추가 비용이 없다. 많은 결과를 모을 때는 `POST` 1회 + `GET` N회 패턴이 유리하다.

> ⚠️ `searchKey`는 검색 조건만 저장하고 결과를 스냅샷하지 않는 것으로 보인다. 매물이 팔리거나 내려가면 페이지를 넘기는 사이에도 `total` 값이 변할 수 있다. (예: `POST` 시점 `total: 395` → 잠시 후 `GET` 시점 `total: 391`) 오래된 키는 만료되어 오류가 날 가능성이 있다(확인 필요).

## 요청 (Request)

### 경로 파라미터 (Path)

| 필드 | 타입 | 필수 | 설명 |
| --- | --- | :---: | --- |
| `searchKey` | `string` (UUID) | ✅ | `POST` 응답으로 받은 검색 세션 키 |

### 쿼리 파라미터 (Query)

| 필드 | 타입 | 필수 | 설명 |
| --- | --- | :---: | --- |
| `accountId` | `number` | ✅ | 요청 계정 ID |
| `characterId` | `number` | ✅ | 요청 캐릭터 ID |
| `page` | `number` | ✅ | 페이지 번호 (1부터 시작) |
| `limit` | `number` | ✅ | 페이지당 아이템 수. 예: `20` |
| `sortType` | `string` | ✅ | 정렬 기준. [정렬 타입](./아이템-검색-tooltip.md#정렬-타입-sorttype) 참고. 재검색 없이 정렬만 바꿔 조회 가능 |

### 요청 예시

```
GET /v1/market/web/items/searches/5bbf4bbc-ab0b-4678-a142-b96a3f3390ac/tool-tip
      ?accountId=12345678
      &page=2
      &limit=20
      &sortType=PRICE_PER_ITEM_ASC
      &characterId=87654321
```

## 응답 (Response)

응답 구조는 [아이템 검색 (툴팁 포함)](./아이템-검색-tooltip.md#응답-response)의 `POST` 응답과 **동일**하다. 최상위에 `items` 배열과 페이지네이션 필드(`page`, `limit`, `total`, `totalPages`, `hasNext`, `searchKey`)가 담긴다. 각 `Item` 객체와 `toolTip` 구조도 그대로다.

### 최상위 구조

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `items` | `Item[]` | 해당 페이지의 매물 목록. [Item 객체](./아이템-검색-tooltip.md#item-객체) 참고 |
| `page` | `number` | 현재 페이지 번호 |
| `limit` | `number` | 페이지당 아이템 수 |
| `total` | `number` | 전체 검색 결과 수 (조회 시점 기준, 변동 가능) |
| `totalPages` | `number` | 전체 페이지 수 (`= ceil(total / limit)`) |
| `hasNext` | `boolean` | 다음 페이지 존재 여부 |
| `searchKey` | `string` (UUID) | 이번 검색 세션 키 (경로에 넘긴 값과 동일) |

---

## 페이지네이션 흐름

1. `POST .../searches/tool-tip` 로 검색 → 응답에서 `searchKey`와 1페이지 결과를 받는다.
2. `hasNext`가 `true`인 동안 같은 `searchKey`로 `GET .../searches/{searchKey}/tool-tip` 을 호출하며 `page`를 증가시킨다.
3. 정렬만 바꾸려면 재검색(`POST`) 없이 `sortType`만 바꿔 `GET`을 호출한다.
