# 응답 예시 — 풀옵션 아이템 1건

`search_items`가 **AI에게 실제로 돌려주는 문자열**의 예시. 아이템이 1건이고, 소울 · 익셉셔널 · 스타포스 · 주문서 · 잠재 · 에디셔널 · 세트효과 · 추가옵션이 **모두** 붙은 최대치 상황을 가정한다.

> 이 문서의 JSON은 가공한 게 아니라, 아래 픽스처를 실제 `summarizeItem()` 코드에 통과시켜 얻은 출력이다.

`search_items`(POST)는 가격 낮은순 10개 + `searchKey`를 반환하고, `get_page`(GET)는 그 `searchKey`로 정렬·페이지를 바꿔 조회한다. **매물(`items[]`)은 두 도구 모두 아래 압축된 요약 형태**다. (원본 툴팁은 매물당 50+ 필드라 토큰이 커서, AI에게는 값 판단에 필요한 핵심만 추려서 준다.)

최상위는 페이지네이션 메타 — `total`(전체 매물 수), `page`, `limit`, `totalPages`, `hasNext`(다음 페이지 존재), `searchKey`.

---

## 1. 요약 응답 — AI가 받는 것

`get_page({ searchKey, page: 1, limit: 20, sort: "PRICE_PER_ITEM_ASC" })` 응답에서 매물 1건이 아래처럼 온다 (그 매물에 소울·익셉셔널·스타포스·주문서·잠재·에디셔널·추옵이 모두 붙은 최대치 가정):

```json
{
 "total": 688,
 "page": 1,
 "limit": 20,
 "totalPages": 35,
 "hasNext": true,
 "searchKey": "7ff4bcdb-5fd4-47ec-9280-cfe11ec3bf8c",
 "items": [
  {
   "id": "Zx9Kq2mPvL7nHwT4bQ8sYcRfEjA1dGhUoI3pN6uW0e:0",
   "name": "제네시스 완드",
   "price": 15000000000,
   "quantity": 1,
   "starforce": 22,
   "scroll": "강화 8회 (남은 횟수 0 / 복구가능 1 / 총 9)",
   "powerDiff": 12500,
   "finalStat": {
    "str": 0, "dex": 0, "int": 350, "luk": 100, "all": 12,
    "pad": 0, "mad": 285, "mhp": 1800, "dam": 10, "bdr": 35, "imdr": 20
   },
   "exOption": "마력 +33 / INT +40 / 데미지 +6% / 올스탯 +9%",
   "potential": "레전드리: 마력 +12% / 몬스터 방어율 무시 +40% / INT +9%",
   "additional": "레전드리: 마력 +9% / 크리티컬 데미지 +4% / 마력 +6%",
   "exceptional": "INT +30 / 마력 +15",
   "soul": "위대한 데미안의 소울 / 공격력 +3%",
   "tradeDesc": "1회 교환 가능 (거래 후 교환 불가) · (가위: 7 / 10)",
   "endDate": "2026-07-10T08:15:00.000Z",
   "wishlist": 47
  }
 ]
}
```

### 요약 필드 설명

| 필드 | 의미 |
| --- | --- |
| `id` | 매물 고유 ID (`{tradeSn}:{subIdx}`) |
| `name` | 아이템 이름 |
| `price` | **개당 가격**(메소). 총가는 `price × quantity` |
| `quantity` | 수량 |
| `starforce` | 스타포스 강화 수치 |
| `scroll` | 주문서 강화 문자열 — `"강화 N회 (남은 횟수 x / 복구가능 y / 총 z)"`. `총 z = 강화 + 남은 + 복구가능` |
| `powerDiff` | **전투력 증가량**(이 아이템 착용 시 현재 장비 대비 증감). **음수 가능**(현재 장비보다 낮으면 마이너스). 캐릭터가 **착용 가능한** 아이템에만 채워지고, 착용 불가(다른 직업 무기 등)면 `null` |
| `finalStat` | 최종 합산 스탯. **고정 키, 0도 표기.** str/dex/int/luk, all(올스탯%), pad(공격력), mad(마력), mhp(최대HP), dam(데미지%), bdr(보스공%), imdr(방무%). *방어력·MP는 제외* |
| `exOption` | 추가옵션 항목 (예: `"마력 +33 / INT +40 / ..."`) |
| `potential` | 잠재능력 — `"등급: 옵션 / 옵션 / ..."` (등급명만, `"잠재능력 :"` 접두어 없음) |
| `additional` | 에디셔널 잠재능력 (동일 형식) |
| `exceptional` | 익셉셔널 강화 내역 |
| `soul` | 소울 — `"이름 / 옵션"` (미장착이면 `null`) |
| `tradeDesc` | 거래 관련 설명 — 교환 가능 여부 + 가위(교환) 잔여 횟수. `"가위 사용 잔여 횟수 :"`는 `"가위:"`로 축약. 예: `"1회 교환 가능 (거래 후 교환 불가) · (가위: 7 / 10)"` |
| `endDate` | 판매 등록 만료 일시 (UTC) |
| `wishlist` | 찜한 사람 수 |

### 요약이 버리는 것 (상세에만 있음)

| 실제 아이템에 있음 | 요약에 있나? |
| --- | :---: |
| **세트효과 이름** | ❌ 상세에만 |
| **공격속도 · 착용직업 · 카테고리** | ❌ 상세에만 |
| **강화 상세(스타포스/주문서 설명 텍스트)** | ❌ 상세에만 |
| **소울 스킬명** | ❌ 상세에만 (요약은 소울명·옵션만) |
| 부여 스킬 · 방어력 · MP | ❌ 상세에만 |

> ⚠️ **황금망치 사용 여부**는 이 API가 제공하지 않는다. 주문서 슬롯 총량(`총 z`)에 이미 합산돼 있어, 아이템 기본 슬롯 수 없이는 분리할 수 없다. (실측: 아케인셰이드 가즈 60건 모두 `max=9` 고정)
