# 캡틴(챌린저스) 스탯 재구성 — 완료 (13/13 잔차 0)

`docs/stat-residuals.md`의 방법론(소스 특정 → 잔차 0)을 캡틴/챌린저스로 확장.
검증 캐릭터: **꽈숩노**(캡틴, 챌린저스, Lv281). 형이 검증한 카데나·보우마스터는 일반서버라
챌린저스 전용 소스가 엔진에 없었음.

## 결과: 13개 항목 전부 잔차 0 ✅ (끼워맞춘 값 없음)
```
STR · DEX · INT · LUK · 데미지 · 보스뎀 · 상태이상뎀 · 방무 · 크확 · 크뎀 · 최종뎀 · 공격력 · 마력
```

## ⚠️ 교훈 (반드시 지킬 것)
- **잔차 0을 억지로 맞추지 말 것.** 한때 파이렛 블레스를 올스탯 90(3중첩)으로 "끼워맞춰" 주스탯 0을 만들었으나,
  이는 실제로 **미발견 소스(하이퍼 버닝 캐시장비)를 가리고 있었다.** 파이렛을 실제 값(1중첩 30)으로 바로잡고
  캐시장비를 API로 받으니 전부 저절로 0으로 닫혔다.
- **정확한 값 > 잔차 0.** 값이 불확실하면 추정/맞춤 대신 유저(도메인 전문가)에게 실측을 물어본다.

## 해소한 소스 (전부 실측/API — 끼워맞춤 없음)

| 소스 | 값 | 출처 |
|---|---|---|

| 캡틴 직업 패시브 | 크확60·크뎀50·최종뎀62%·공100·주스탯·방무20 | 스킬 툴팁 각각 파싱 (jobPassive.ts) |
| 도전자 세트 팬텀 8세트 | 7세트 캡 | 넥슨 API 버그(total_set_count 8+복제 8세트). 세트창 7/7 실측 |

| 소비 불릿(초보 해적의 불릿) | 공10 | 툴팁 실측 (API 미노출, 불릿 종류마다 다름 → 하드코딩) |

| **하이퍼 버닝 캐시장비** | 소드 공10마10 · 로브/크라운 올스탯20 | **`/character/cashitem-equipment` API** (신규 조회 추가) |
| 여제/정령 축복 중복 버그픽스 | 높은 레벨만(blessSkip) | 여제 툴팁 "정령과 비교 높은 레벨만 적용" |
| 쓸만한 어드밴스드 블레스 | 공/마 20 (cond 제거) | 지속시간 없는 상시 패시브(유저 확인) |

## 소스별 API 조회 가능 여부 (일반화 관점)

### ✅ API로 값까지 완전 자동
| 소스 | 엔드포인트 |
|---|---|
| 장비·심볼·하이퍼·어빌·링크·유니온·성향·스킬·HEXA | (형 기존) |
| **길드 노하우** (상시 공/마) | `/guild/id` + `/guild/basic` (신규) |
| **하이퍼 버닝 캐시장비** (소드/로브/크라운) | `/character/cashitem-equipment` (신규) |

> 길드는 액티브 30분 버프(보스킬링머신 등)·일반몹뎀 제외하고 상시 노하우만.

### 🔶 존재 여부만 API, 값은 하드코딩 — 근데 **모든 해당 캐릭 동일(고정값)**
스킬창에 수치가 안 떠서(/skill에 없음) 값은 하드코딩하되, 같은 조건이면 누구나 같은 값:
| 소스 | 게이팅 | 값 |
|---|---|---|
| 챌린저스 서버버프 | `world_name==='챌린저스'` | 서버 전원 동일 고정 |
| 버닝 BEYOND / 하이퍼버닝 MAX | 스킬 보유 | 레벨1 고정 |
| 캡틴 직업 패시브 | 직업=캡틴 | 직업별 상수(만렙 가정, 형 정책) |

### 🔴 하드코딩 + **캐릭터마다 값이 다름** (config/입력 필요)
같은 조건이어도 캐릭별로 값이 달라 자동화 불가 — 현재 꽈숩노 값 하드코딩:
| 소스 | 게이팅 | 왜 캐릭별 |
|---|---|---|
| 찬란한 결계의 핵 | "결계의 핵 소환" 스킬 | 등급(유니크 등)·+강화마다 스탯 상이 |
| 소비 불릿 | "인피닛 불릿" 스킬 | 불릿 종류마다 공격력 상이(초보=10) |
| 파이렛 블레스 중첩수 | 본인 해적 링크 | 월드 내 서로다른 모험가 해적 직업군 수(1~3) |

→ 🔴 항목은 넥슨 API에 없음. **캐릭터별 수동입력(config)** 또는 `/character/stat` final_stat을
   base로 신뢰하고 재구성 검증에서만 예외 처리하는 방식 필요. (형과 설계 상의)

## 회귀 확인 필요 (형 검토)
- **여제/정령 blessSkip**, **어드밴스드 블레스 cond 제거**, **길드/캐시 신규 조회** — 공통 로직이라
  오유찬/티엘 등 검증 직업 재구성에도 반영됨. 잔차 0 유지되는지 재확인.
