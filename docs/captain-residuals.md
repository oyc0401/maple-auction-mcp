# 캡틴(챌린저스) 스탯 재구성 — 진행 상황

`docs/stat-residuals.md`의 방법론(소스 특정 → 잔차 0)을 캡틴/챌린저스로 확장.
검증 캐릭터: **꽈숩노**(캡틴, 챌린저스, Lv281). 형이 검증한 카데나·보우마스터는 일반서버라
챌린저스 전용 소스가 엔진에 없었음.

## 현재 상태: 13개 항목 중 11개 잔차 0 ✅

```
✅ 잔차 0:  STR · DEX · INT · LUK · 데미지 · 보스뎀 · 상태이상뎀 · 방무 · 크확 · 크뎀 · 최종뎀
⚠️ 남음:   공격력 ~3% · 마력 ~5%  (아래 "미해결" 참고)
```

주스탯 4개 전부 잔차 0 달성. 딜의 핵심 스탯은 완결.

## 해소한 챌린저스/캡틴 전용 이슈

| 소스 | 처리 | 검증 |
|---|---|---|
| **챌린저스 서버 상시버프** | `collectChallenger` (`world_name==='챌린저스'` 게이팅). 올100·공마80·보공70·방무70·크확30·크뎀40 | 크확/보스뎀 잔차 0 |
| **버닝 BEYOND / 하이퍼 버닝 MAX** | `collectBurning`. 옵트인이라 스킬 보유로 게이팅. 상호배타. 올30·공마30·보공20·방무20. **"스킬창 미표시" → API에 수치 없어 툴팁 실측 하드코딩** | 보스뎀·방무 잔차 0 |
| **캡틴 직업 패시브** | `JOB_PASSIVES['캡틴']` (jobPassive.ts). 1~4차 무조건 상시: 크확60·크뎀50·최종뎀62%·공100·주스탯·방무20 | 크확/크뎀/최종뎀 잔차 0(소수점 일치) |
| **도전자 세트 팬텀 8세트** | `SET_MAX_COUNT`로 7세트 캡. 넥슨 API가 total_set_count 8 + 6세트 복제한 팬텀 8세트(방무10%)를 주는 버그. 인게임 세트창 7/7 실측 | 방무 잔차 0 |
| **파이렛 블레스(본인 해적 링크)** | `LINK['파이렛 블레스']` 룰 수정 + 직업 접미사("(캡틴)") 매칭. 올스탯30×3중첩=90. 챌린저스는 타직업 링크 불가, 본인 링크만 적용 | INT/LUK 잔차 0 |
| **찬란한 결계의 핵** | 장착형 상시 아이템. **API 미노출**(스킬 "결계의 핵 소환"만 있음), 등급·+강화마다 값 상이. 꽈숩노(유니크+7): DEX65·STR65·공40·DEX%20. `nexon.ts`에 스킬 게이팅+하드코딩 | STR/DEX 잔차 0 |
| **여제/정령 축복 중복 버그 픽스** | `blessSkip` — 여제·정령은 중복 불가, 높은 레벨(공격력)만 적용. 기존 COMMON은 둘 다 합산(공/마 각 +15 과다) | (아래 회귀확인 필요) |
| **쓸만한 어드밴스드 블레스** | 지속시간 없는 상시 패시브 — cond 제거해 공/마 20 포함(유저 확인) | 공/마 잔차 축소 |

## ⚠️ 미해결: 공격력/마력 (~3%/~5%)

버프(컨티뉴어스 링·영웅의 메아리·펫) 다 끄고 재조회해도 **공격력 -99·마력 -55** 남음.
- **초보 해적의 불릿(공10)** — 소비창 장착 불릿. 인피닛 불릿으로 상시 적용. **API 미노출 + 불릿 종류마다 값 상이**(결계와 동일 성격). 게임 스탯창 "소비 아이템 10"의 정체.
- 넥슨 stat API 갱신 지연 + 시간제 % 버프(영메 4%·컨티뉴어스 8%) 잔재 → 완벽 순수상태 캡처가 어려움.
- **결론:** 공격력은 D 공식에서 모든 매물 공통 배수라 매물 비교(D_after/D_before)에선 거의 상쇄. 절대 환산이 아닌 상대 비교 목적이면 실용상 영향 미미.

## API 미노출 변동 소스 (설계 과제)
버닝BEYOND·결계의 핵·불릿 — 3개 모두 **넥슨 API에 항목 없음 + 캐릭터마다 값 상이**. 현재 하드코딩(꽈숩노 값). 일반화하려면:
- (안1) 캐릭터별 수동입력(config)
- (안2) `/character/stat`의 final_stat은 이미 이것들을 포함하므로, 재구성 대신 final_stat을 base로 신뢰하고 이 소스들은 재구성 검증에서만 예외 처리
→ 형과 상의할 설계 포인트.

## 회귀 확인 필요 (형 검토)
- **여제/정령 blessSkip**: 여제+정령 둘 다 가진 **모든 직업** 영향(오유찬/티엘 포함). 기존 잔차 0이었다면 이 15 과다가 다른 데서 상쇄됐거나 공격력을 엄격히 안 봤다는 뜻 — 재확인 필요.
- **어드밴스드 블레스 cond 제거**: 공통 COMMON 변경 → 타 직업 공/마 +20 영향.
- **파이렛 블레스 mult:3**: 스택 수가 API에 없어 최대 3 하드코딩 — 계정별 해적 수에 따라 다름.
