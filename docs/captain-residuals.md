# 캡틴(챌린저스) 스탯 재구성 — 완료 (13/13 잔차 0)

`docs/stat-residuals.md`의 방법론(소스 특정 → 잔차 0)을 캡틴/챌린저스로 확장.
검증 캐릭터: **꽈숩노**(캡틴, 챌린저스, Lv281). 형이 검증한 카데나·보우마스터는 일반서버라
챌린저스 전용 소스가 엔진에 없었음.

## 결과: 13개 항목 전부 잔차 0 ✅ (끼워맞춘 값 없음)
```
STR · DEX · INT · LUK · 데미지 · 보스뎀 · 상태이상뎀 · 방무 · 크확 · 크뎀 · 최종뎀 · 공격력 · 마력
```

## ⚠️ 교훈 (반드시 지킬 것)
- **잔차 0을 억지로 맞추지 말 것.** 한때 파이렛 블레스를 올스탯 90(3중첩)으로 "끼워맞춰" 주스탯 0을 만들었으나,
  이는 실제로 **미발견 소스(하이퍼 버닝 캐시장비)를 가리고 있었다.** 파이렛을 실제 값(1중첩 30)으로 바로잡고
  캐시장비를 API로 받으니 전부 저절로 0으로 닫혔다.
- **정확한 값 > 잔차 0.** 값이 불확실하면 추정/맞춤 대신 유저(도메인 전문가)에게 실측을 물어본다.

## 신규 추가한 넥슨 API 엔드포인트
- `/guild/id` + `/guild/basic` — 길드 노하우(상시 공/마 패시브). 액티브 30분 버프·일반몹뎀은 제외.
- `/character/cashitem-equipment` — 하이퍼 버닝 등 능력치 캐시장비.

## 회귀 확인 필요 (형 검토)
- **여제/정령 blessSkip**, **어드밴스드 블레스 cond 제거** — 공통 COMMON 변경 → 오유찬/티엘 등 검증 직업 재확인.
- **길드/캐시 신규 조회** — 다른 직업 재구성에도 반영됨(정상). 잔차 0 유지되는지 확인.

## 캐릭터별 하드코딩 (API 미노출 — 추후 config/입력화)
- 결계의 핵(등급/+강화별 상이), 불릿(종류별 상이), 파이렛 블레스 중첩수(월드 해적직업 수) — 꽈숩노 값으로 하드코딩/게이팅.
- 일반화하려면 캐릭터별 수동입력 또는 final_stat 신뢰 방식. (형과 상의)
